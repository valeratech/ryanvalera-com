# ADR-003 — GitHub Pages as Primary Origin

## Status

Accepted

## Date

June 2026

---

## Context

The platform requires a primary origin to serve the static site. The primary origin must support HTTPS natively to satisfy Cloudflare SSL/TLS Full Strict mode.

---

## Decision

Use GitHub Pages as the primary origin.

---

## Rationale

### Cost

GitHub Pages is free for public repositories.

### Reliability

GitHub Pages is a mature, highly available static hosting service with a strong uptime track record. No operational overhead is required to maintain it.

### HTTPS Native

GitHub Pages provides a valid TLS certificate for custom domains automatically once DNS is configured. This satisfies Cloudflare SSL/TLS Full Strict mode without additional certificate management.

### CI/CD Integration

GitHub Pages deploys automatically on push to `main` from the same repository that hosts the source code — no separate deployment pipeline is required for basic static site delivery. A GitHub Actions workflow will be added later to extend this with cache invalidation and Cloudflare Pages synchronization.

### Direct Reachability

GitHub Pages is independently reachable at `https://valeratech.github.io/ryanvalera-com/` even if Cloudflare DNS is bypassed, providing a genuine fallback URL during Cloudflare platform events.

---

## Consequences

- GitHub Pages is the primary traffic destination under normal operation
- GitHub Pages outages trigger automatic failover to Cloudflare Pages via Cloudflare Load Balancer health checks
- Response procedure for GitHub Pages outages is documented in `docs/runbooks/origin-failure.md`
