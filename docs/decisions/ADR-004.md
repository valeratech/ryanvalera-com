# ADR-004 — Cloudflare Load Balancing

## Status

Accepted

## Date

June 2026

---

## Context

This is a low-traffic static portfolio site. A single origin (GitHub Pages) is sufficient to serve it reliably. Cloudflare Load Balancing is not required for availability or performance at this traffic level.

---

## Decision

Implement Cloudflare Load Balancing with two origin pools (GitHub Pages as primary, Cloudflare Pages as secondary) and failover steering.

---

## Rationale

The purpose of this project is explicitly to demonstrate infrastructure engineering concepts at a professional level using a static site as a lightweight, low-cost payload.

Cloudflare Load Balancing was implemented to demonstrate:

- Origin pool configuration
- Health check setup (HTTPS GET, status code validation, response time monitoring)
- Failover steering policy
- Multi-origin traffic engineering
- Live failover behavior (deliberately testable by disabling the primary origin)

This is not an operational necessity for a portfolio site. It is a deliberate portfolio engineering decision — the site exists to carry the infrastructure, not the other way around.

---

## Design Constraint

The CDN-on-CDN conflict that would arise from routing Cloudflare Load Balancer traffic through AWS CloudFront was a real technical constraint evaluated during origin selection. The decision to use Cloudflare Pages as the secondary origin (rather than AWS CloudFront + S3) directly resolved this constraint. See ADR-002.

---

## Interview Framing

When discussing this architecture, the correct framing is:

> "The goal wasn't scaling traffic. The goal was demonstrating origin pool management, health monitoring, and failover logic using Cloudflare Load Balancing in a real deployed environment under a strict zero-cost constraint."

---

## Consequences

- Cloudflare Load Balancing incurs a monthly cost (~$5/month base + usage)
- Both origins must maintain valid HTTPS certificates (Full Strict requirement)
- Health check failures on GitHub Pages trigger automatic failover to Cloudflare Pages
- Failover test procedure is documented in `docs/runbooks/failover-test.md`
