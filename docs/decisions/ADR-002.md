# ADR-002 — Cloudflare Pages as Secondary Origin

## Status

Accepted

## Date

June 2026

---

## Context

The platform requires a secondary origin for Cloudflare Load Balancer failover. Several options were evaluated:

| Option | SSL/TLS | Proxy Chaining | Assessment |
|---|---|---|---|
| AWS CloudFront + S3 | Requires CloudFront in front of S3 | CDN-on-CDN conflict | Rejected |
| AWS S3 direct | HTTP only — no native HTTPS | Forces Flexible SSL mode | Rejected |
| Cloudflare Pages | HTTPS native | No proxy conflict (internal routing) | Selected |
| Netlify | HTTPS native | Minor proxy chain | Viable alternative |

---

## Decision

Use Cloudflare Pages as the secondary origin for the Cloudflare Load Balancer.

---

## Rationale

### SSL/TLS Compatibility

Cloudflare SSL/TLS Full Strict mode requires valid certificates at the origin. Cloudflare Pages provides a valid TLS certificate natively, satisfying Full Strict without additional configuration.

AWS S3 static website endpoints are HTTP-only, which would require Flexible SSL mode — an unacceptable security downgrade.

### No Proxy Chaining

Cloudflare Load Balancer routing to Cloudflare Pages is essentially internal routing — there is no CDN-on-CDN conflict, no SSL termination conflict, and no risk of redirect loops or cache header conflicts.

The alternative (Cloudflare in front of AWS CloudFront) was evaluated and produced SSL/TLS errors in testing, consistent with known behavior when layering two CDN/proxy systems.

### Independent Deployment Pipeline

Cloudflare Pages connects directly to the same GitHub repository (`valeratech/ryanvalera-com`) and deploys automatically on push to `main` — independently of the GitHub Pages deployment. This provides genuine deployment independence between the two origins.

### Cost

Cloudflare Pages is free at the usage levels this project requires.

---

## Consequences

- Both origins are served through or by Cloudflare infrastructure (see ADR-001)
- A full Cloudflare platform event affects both origins simultaneously
- GitHub Pages remains independently reachable without Cloudflare
